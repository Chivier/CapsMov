# Product

## Register

product

## Users

macOS users who prefer keyboard-driven navigation and want a lightweight way to reuse Caps Lock as a temporary movement layer without installing a larger remapping suite. They interact with CapsMov briefly through the menu bar, primarily to confirm status, grant permissions, adjust behavior, or recover from a blocked input state.

## Product Purpose

CapsMov turns Caps Lock into a dependable navigation layer while preserving the key's normal tap behavior. Success means the remapping works quietly in the background, setup and recovery are understandable, and users can verify the app's state without learning implementation details.

## Brand Personality

Native, restrained, trustworthy. The voice is direct and calm, especially when explaining permissions, Secure Input, updates, and actions that affect the current macOS session.

## Anti-references

CapsMov should not resemble a decorative dashboard, a highly themed keyboard utility, or an enterprise settings console. Avoid flashy visual effects, custom replacements for familiar macOS controls, dense configuration surfaces, and technical error messages without a practical next step.

## Design Principles

1. Stay out of the way until the user needs status or recovery guidance.
2. Make the current state and next safe action immediately clear.
3. Prefer familiar macOS controls and system language over invented interaction patterns.
4. Keep advanced implementation details out of the primary workflow.
5. Be precise and honest about permissions, update behavior, and system limitations.

## Accessibility & Inclusion

Follow native macOS accessibility conventions. Support VoiceOver labels, keyboard operation, clear text contrast, and status communication that does not rely on color alone. Respect reduced-motion preferences and avoid decorative animation.
